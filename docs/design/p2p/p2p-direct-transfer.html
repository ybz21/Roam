部署与使用 · 客户端就是一个浏览器

开发机部署一套 Roam → frp 穿透到公网。客户端只需一个浏览器打开公网 https:// frp 地址，任意地点 / 任意数量、零安装。P2P 是透明升级，入口不变。
① 入口 · 信令 · 兜底
浏览器 ──▶ 公网 frp 地址
（加载页面 + 交换 SDP/ICE + 打洞失败时中转）
永远走 frp
② 重流量数据面
文件 / 镜像 ──▶ STUN · IPv6 · UPnP 打洞后
点对点直连
★ 绕开 frps，省带宽 ★
关键区分：「穿透 frp 地址」= 入口与信令（理解正确，永远走 frp）；P2P 数据不是穿透 frp，而是绕开 frp 直连 —— frp 只牵线，牵上后两端自己直连；打不通自动退回 frp，同一个浏览器全程无感。
前提 1 · 入口必须 https
WebRTC / 麦克风 / 剪贴板只在安全上下文可用；Roam frp 部署默认开自签 HTTPS，frp 地址须为 https。
前提 2 · 客户端零配置 ≠ 服务端零配置
吃到 P2P 加速要在服务端 / frps 侧一次性配置 STUN（+ 可选 IPv6 / UPnP）；客户端仍什么都不用装。

跨网穿透 Playbook（零成本 · 不建 TURN）

1IPv6 直连zero-cost
两端都有公网 IPv6 时无需打洞，显著提高成功率，专治 IPv4 侧 CGNAT / 对称 NAT。注意 IPv6 防火墙仍可能拦截入站 UDP，非必成。
2UPnP / NAT-PMP 映射zero-cost
端口必须一致：SetNAT1To1IPs 只改 IP 不改端口。先固定 UDP 端口 + UDPMux，映射同一端口；external==internal 才宣称 candidate，否则跳过。M0 实测。
3STUN 架在 frps 上zero-cost
现成公网 IP、流量≈0、免依赖 Google。对全锥 / 端口受限锥 NAT 有效。
4手动 UDP 端口转发zero-cost
能转发时显著提高成功率（上游 CGNAT / 防火墙仍拦则无效）。文档给步骤 + 固定端口。

先弹 showSaveFilePicker─▶ negotiating

写入同一 handle
回退即：拆 PC/DC + 发 p2p/cancel 停后端读取 + 置终结标志，忽略迟到的 p2p/connected／数据帧。速率用 goodput（落盘字节增量），candidate-pair 字节/RTT 仅作诊断。

关键决策

  • 纯增益、零新增风险：P2P 建不起来就全体走 frp，行为等同现状；按服务通道独立兜底。
  • picker 先于协商：showSaveFilePicker() 必须在点击的用户激活窗口内调用；回退写入同一 handle，绝不二次下载。
  • DataChannel 是消息通道（非字节流）：一条 message = 一帧，16 KiB 保守基线。
  • MVP 每下载一个 PC：共享 PC 涉及 SCTP 关联拥塞共享、大文件冲击实时流，须 ROI 后专门压测再做。
  • i18n 强制：所有连接状态 / 错误 / 按钮文案走全局 i18n，zh-CN + en-US 双份。
  • 诚实下限：服务器纯 CGNAT-IPv4、无 IPv6、不能转发 → 打洞失败回退 frp，零成本无解（拓扑硬约束）。
